Lunamux config locations & wipe map

Termtastic → Lunamux  ·  macOS Electron app

Where the config lives — and what the wipe script does with it

Everything the desktop app has ever persisted, across three naming schemes, mapped to who reads/writes it and what scripts/delete-all-config.sh does to each. Run it to test a clean Termtastic install → Lunamux upgrade from a blank slate.

The one thing to understand

The rename deliberately did not relocate existing installs. Prod keeps its userData dir, its SQLite dir, its bundle id (se.soderbjorn.termtastic), and its shared-settings filename (termtastic.json) pinned to the old names so an upgrade never orphans a user's data. Only dev launches, named instances, the demo, logs, and the productName/package.json name-derived defaults follow the new Lunamux name. That split is why config is scattered across ~a dozen directories.

Bug found & fixed during this audit

The rename pinned prod's Electron userData to the wrong directory. The old packaged app resolved app.getName() from the asar's name field (termtastic-electron, no productName) → its real client state (device token, cookies, local_state.json, dismissed news) lived in …/termtastic-electron. The rename pinned to …/Termtastic instead — a separate dir that only ever held the JVM server's SQLite DB — which orphaned every install's client state on upgrade (proof: the post-rename Termtastic/local_state.json is a blank 38-byte stub).

Fix applied: prod now pins to appData/termtastic-electron, restoring the clean split (client → termtastic-electron, server DB → Termtastic). No migration needed — the original state is still there. lunamux-electron was always empty; nothing used it.

01

Electron userData directories

~/Library/Application Support/

The heavy hitters. Each holds the SQLite databases, the device-auth token (local_state.json), the TLS keystore (tls/), generated shell-init/, the electron-*.json window/session caches, and Chromium browser state. Written by the Electron main (path pinning), the bundled JVM server (SQLite + TLS + shell-init), the renderer's LocalStore, and Chromium. The script renames each whole dir to <name>.bak.<timestamp>.

02

Package-name userData directories

~/Library/Application Support/ — lowercase, hyphenated

appData/<name> is Electron's default userData, where name is electron/package.json's name field. The packaged prod app resolves here (no productName in the asar), so termtastic-electron holds the real prod client state — device token, cookies, IndexedDB, dismissed news. After the fix above, prod pins back to termtastic-electron; lunamux-electron stays empty. The old script missed all three entirely.
